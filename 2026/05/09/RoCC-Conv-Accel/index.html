项目概述

基于 RISC-V RoCC (Rocket Custom Coprocessor) 接口的 5×5 滑动窗口矩阵卷积加速器。使用 Chisel 编写，Verilator 仿真验证。

  • 仓库：MyConvAccel
  • 当前进度：36 个测试全部通过
参数 值
输入矩阵 32×32
最大卷积核 5×5
数据格式 Q8.8 定点数（16-bit）
累加器 32-bit
输出 同尺寸 zero-padding
性能目标 <2500 周期，≥40x 加速

Phase 1：RoCC 指令调度基础设施

搭建 RoCC custom0 指令调度框架，让 Rocket Core 能够向加速器发送指令。

funct7 指令 描述
0 SET_ADDR_IN 设置输入矩阵基地址（8 字节对齐）
1 SET_ADDR_KER 设置 5×5 卷积核基地址（2 字节对齐）
2 SET_ADDR_OUT 设置输出矩阵基地址（8 字节对齐）
3 START_ACCEL 非阻塞启动加速器
4 POLL_STATUS 查询状态（busy/done/overflow/addr_err）

Phase 2：DMA 地址对齐与异常检测

实现 DMA 地址设置与对齐检查，非法地址触发 addr_err 并中止。

地址类型 对齐要求
ADDR_IN 8 字节
ADDR_KER 2 字节
ADDR_OUT 8 字节

状态寄存器 bit 位：

  • bit[0] — Busy（加速器运行中）
  • bit[1] — Done（计算完成）
  • bit[2] — Overflow（计算中出现饱和）
  • bit[3] — Addr_Err（地址未对齐或为空）

Phase 5：定点数饱和处理

MAC 树输出右移 8 位后饱和到 16-bit 有符号范围。

  • 上溢 → 输出 0x7FFF（32767）
  • 下溢 → 输出 0x8000（-32768）
  • 正常 → 输出低 16 位
